## 2. 에러 원인 분석

### 증상 흐름

```
1. 사용자가 "HTS 조건식 불러오기" 클릭
2. 서버가 condition.list 작업을 에이전트 큐에 삽입
3. 에이전트가 kiwoom_ws_request("ka10171", {"trnm": "CNSRLST"}) 호출
4. Python websocket-client로 wss://api.kiwoom.com:10000/api/dostk/websocket 연결
5. ← 이 시점에서 헤더(Authorization, api-id)가 서버에 전달되지 않음
6. 키움 서버: 인증 없는 클라이언트로 판단
7. on_open 즉시 {"trnm": "CNSRLST"} 전송
8. 키움 서버: 에러 100013 반환
```

### 직접 원인: Python `websocket-client`의 딕셔너리 헤더 미전달

현재 코드 (189~192번 줄):

```python
ws_app = websocket.WebSocketApp(
    ws_url,
    header={                              # ← 딕셔너리 형식
        "api-id": api_id,
        "Authorization": f"Bearer {token}",
    },
    ...
)
```

`websocket-client` 라이브러리의 `WebSocketApp`은 `header` 파라미터를 **딕셔너리**로 받을 수 있지만, 실제 HTTP Upgrade 요청 헤더로 포함되지 않는 버전이 있다.

반면 **리스트 형식**(`["key: value", ...]`)으로 전달하면 HTTP Upgrade 요청에 확실하게 포함된다. 이것이 `websocket-client` 라이브러리의 표준 헤더 전달 방식이다.

### 비교: Node.js 기존 코드 vs Python 에이전트

| 항목 | Node.js (`kiwoom.condition.ts`) | Python 에이전트 (현재) |
|------|--------------------------------|----------------------|
| 라이브러리 | `ws` | `websocket-client` |
| 헤더 키 | `authorization` (소문자) | `Authorization` (대문자) |
| 헤더 전달 방식 | `headers: { ... }` — ws 라이브러리가 HTTP Upgrade에 확실히 포함 | `header={ ... }` 딕셔너리 — 실제 전달 불확실 |
| 결과 | 정상 동작 가능성 있음 | 에러 100013 |

## 4. 수정 지시 (소스코드 레벨)

### 수정 위치

`agent/kiwoom-agent.py` **187~196번 줄** — `kiwoom_ws_request` 함수 내 `WebSocketApp` 생성 부분

### 현재 코드 (187~196번 줄)

```python
    ws_app = websocket.WebSocketApp(
        ws_url,
        header={
            "api-id": api_id,
            "Authorization": f"Bearer {token}",
        },
        on_open=on_open,
        on_message=on_message,
        on_error=on_error,
    )
```

### 수정 후 코드

```python
    ws_app = websocket.WebSocketApp(
        ws_url,
        header=[
            f"api-id: {api_id}",
            f"Authorization: Bearer {token}",
        ],
        on_open=on_open,
        on_message=on_message,
        on_error=on_error,
    )
```

### 변경 내용 요약

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| `header` 타입 | `dict` (`{}`) | `list` (`[]`) |
| `api-id` 형식 | `"api-id": api_id` | `f"api-id: {api_id}"` |
| `Authorization` 형식 | `"Authorization": f"Bearer {token}"` | `f"Authorization: Bearer {token}"` |
| 변경 줄 수 | — | 5줄 (189~192번 줄 대체) |

## 5. 수정 후 검증 방법

에이전트를 재시작한 후:

1. 화면에서 **조건검색** 메뉴 이동
2. **"HTS 조건식 불러오기"** 버튼 클릭
3. 기대 결과: 에러 없이 키움 HTS에 등록된 조건식 목록이 드롭다운에 표시됨
4. 에이전트 콘솔 로그에서 `condition.list 조회 완료: N개` 확인

### 검증 실패 시 추가 확인 사항

에러가 계속 100013이라면 헤더 이외에 **WebSocket 연결 후 별도 로그인 메시지**가 필요한 구조일 수 있다. 이 경우 `on_open` 콜백을 아래처럼 2단계로 수정해야 한다:

```python
# 1단계 연결 직후 로그인 먼저 보내고
# 2단계 로그인 응답 받은 후 실제 요청 보내는 구조
def on_open(ws):
    ws.send(json.dumps({
        "trnm": "LOGIN",
        "token": token,
    }))
    # 이후 on_message에서 LOGIN 응답 처리 후 payload 전송

def on_message(ws, raw):
    msg = json.loads(raw)
    if msg.get("trnm") == "LOGIN":
        ws.send(json.dumps(payload))  # 로그인 성공 후 실제 요청
        return
    # ... 기존 응답 처리
```

단, 이 방식이 필요한지는 위 Step 4의 기본 수정 시도 후 에러 여부를 보고 판단한다.
